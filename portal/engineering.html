Context & Prompt Engineering

The core skills for getting high-quality output from AI models.

Part 2: Context Engineering (The Unlock)

Prompt engineering is just the tip of the iceberg. Context Engineering is providing the background materials (PDFs, previous emails, data sets) so the AI actually "knows" what you know.

  • The Goldilocks Principle: Don't dump your entire hard drive. Don't give nothing. Give relevant documents that contain the answers you need.
  • Access to Tools: If you want it to analyze data, give it the spreadsheet. If you want it to summarize a meeting, give it the transcript.
  • Reference Examples: Show, don't just tell. "Write it like this [paste example]" is more powerful than describing the style for 5 paragraphs.

Troubleshooting & Navigation

Path Obsession

Sometimes the AI gets stuck in a loop, trying the same wrong solution over and over. Break the loop. Say "Stop. This isn't working. Let's try a completely different approach."

Context Transfer

Long chats get "stale" and the AI forgets early details. If a chat gets too long/confusing, summarize the key points and start a fresh chat with that summary as the new context.

Task Separation

Don't mix unrelated tasks in one thread. "New Task = New Chat." It keeps the context clean and prevents the AI from getting confused by previous instructions.
